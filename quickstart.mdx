This guide will help you clone, install, and run Sortvis on your local machine. The entire process takes less than 2 minutes.

## Prerequisites

You'll need one of the following package managers installed:

<CardGroup cols={3}>
  <Card title="Bun" icon="b" href="https://bun.sh">
    Recommended - Native package manager
  </Card>
  <Card title="pnpm" icon="p" href="https://pnpm.io">
    Fast, disk space efficient
  </Card>
  <Card title="npm" icon="n" href="https://nodejs.org">
    Comes with Node.js
  </Card>
</CardGroup>

<Note>
  Sortvis uses a `bun.lock` file, meaning Bun is the natively used package manager. However, pnpm and npm work equally well.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Sortvis repository from GitHub:

    ```bash
    git clone https://github.com/qonTesq/sortvis.git
    ```
  </Step>

  <Step title="Navigate to the directory">
    Change into the project directory:

    ```bash
    cd sortvis
    ```
  </Step>

  <Step title="Install dependencies">
    Install packages using your preferred package manager:

    <CodeGroup>
      ```bash Bun
      bun install
      ```

      ```bash pnpm
      pnpm install
      ```

      ```bash npm
      npm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Start the development server">
    Launch the Vite development server:

    <CodeGroup>
      ```bash Bun
      bun run dev
      ```

      ```bash pnpm
      pnpm dev
      ```

      ```bash npm
      npm run dev
      ```
    </CodeGroup>

    You should see output similar to:

    ```bash
    VITE v7.3.1  ready in 234 ms

    ➜  Local:   http://localhost:5173/
    ➜  Network: use --host to expose
    ➜  press h + enter to show help
    ```
  </Step>

  <Step title="Open in browser">
    Navigate to `http://localhost:5173` in your browser to see Sortvis in action.
  </Step>
</Steps>

## Using the visualizer

Once Sortvis is running, you can interact with the visualization:

### Select an algorithm

Choose from 6 sorting algorithms using the algorithm selector in the header:

- **Bubble Sort** - Simple comparison-based algorithm
- **Quick Sort** - Efficient divide-and-conquer with median-of-three pivot
- **Merge Sort** - Stable divide-and-conquer algorithm
- **Heap Sort** - Comparison-based using a binary heap
- **Insertion Sort** - Builds sorted array one element at a time
- **Selection Sort** - Repeatedly finds minimum element

### Adjust array size

Use the size slider to change the number of elements (10-512). Larger arrays make algorithmic differences more visible:

```typescript
const SIZE_MIN = 10;
const SIZE_MAX = 512;
```

### Control playback speed

Choose from 6 speed levels:

<CardGroup cols={3}>
  <Card title="Snail" icon="turtle">
    500ms delay, 1 step/frame - Perfect for learning
  </Card>
  <Card title="Slow" icon="gauge-low">
    100ms delay, 1 step/frame - Educational pace
  </Card>
  <Card title="Medium" icon="gauge">
    10ms delay, 1 step/frame - Default setting
  </Card>
  <Card title="Fast" icon="gauge-high">
    0ms delay, 1 step/frame - Quick visualization
  </Card>
  <Card title="Rapid" icon="forward">
    0ms delay, 5 steps/frame - Very fast
  </Card>
  <Card title="Flash" icon="bolt">
    0ms delay, 100 steps/frame - Near instant
  </Card>
</CardGroup>

The speed configuration is defined in the visualizer engine:

```typescript
const SPEED_CONFIG: Record<SpeedLevel, { delay: number; batch: number }> = {
  snail: { delay: 500, batch: 1 },
  slow: { delay: 100, batch: 1 },
  med: { delay: 10, batch: 1 },
  fast: { delay: 0, batch: 1 },
  rapid: { delay: 0, batch: 5 },
  flash: { delay: 0, batch: 100 }
};
```

### Understand the color codes

Bars change color to indicate their state during sorting:

| Color | State | Description |
|-------|-------|-------------|
| Default | Neutral | Unsorted, not currently being processed |
| Comparing | Active | Being compared with another element |
| Swapping | Action | Being swapped or moved |
| Sorted | Complete | In final sorted position |
| Pivot | Reference | Selected as pivot (Quick Sort) |
| Merge | Action | Being merged (Merge Sort) |

The state types are defined as a discriminated union:

```typescript
type BarState = 'default' | 'comparing' | 'swapping' | 'sorted' | 'pivot' | 'merge';
```

### View statistics

Real-time statistics are displayed at the bottom of the screen (desktop only):

- **Comparisons** - Number of element comparisons performed
- **Swaps** - Number of swaps or array writes executed

```typescript
interface SortStatistics {
  /** Number of comparisons made */
  comparisons: number;
  /** Number of swaps or array writes performed */
  swaps: number;
}
```

## Building for production

Create an optimized production build:

<CodeGroup>
  ```bash Bun
  bun run build
  ```

  ```bash pnpm
  pnpm build
  ```

  ```bash npm
  npm run build
  ```
</CodeGroup>

This generates a static site in the `build/` directory using SvelteKit's static adapter.

Test the production build locally:

<CodeGroup>


  ```bash pnpm
  pnpm preview
  ```

  ```bash npm

  ```
</CodeGroup>

```text
│   │   ├── layout/        # Canvas renderer, Controls, Header, Footer

│   ├── config/        # Configuration files
│   │   ├── algorithms.ts  # Algorithm metadata and complexity info
│   │   └── speeds.ts      # Speed level definitions
│   ├── store/         # State management
│   │   ├── visualizer.svelte.ts  # Main visualizer engine
│   │   └── media.svelte.ts       # Responsive breakpoint detection
│   └── types/         # TypeScript interfaces

└── routes/            # SvelteKit routes
    ├── +page.svelte       # Main application page
    └── +layout.svelte     # Root layout with theme provider
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 5173 is already in use">
    If port 5173 is occupied, Vite will automatically try the next available port (5174, 5175, etc.). Check the terminal output for the actual URL.
    
    You can also specify a custom port:
    
    ```bash
    bun run dev -- --port 3000
    ```
  </Accordion>

  <Accordion title="Module not found errors">
    If you see module resolution errors, try:
    
    1. Delete `node_modules` and lockfile
    2. Reinstall dependencies
    3. Run `bun run prepare` to sync SvelteKit
    
    ```bash
    rm -rf node_modules bun.lock
    bun install
    bun run prepare
    ```
  </Accordion>

  <Accordion title="Type errors in the editor">
    Make sure your editor has the Svelte extension installed and run the prepare script:
    
    ```bash
    bun run prepare
    ```
    
    This syncs SvelteKit's generated types.
  </Accordion>

  <Accordion title="Build fails with memory errors">
    If you encounter out-of-memory errors during build, try increasing Node's memory limit:
    
    ```bash
    NODE_OPTIONS="--max-old-space-size=4096" bun run build
    ```
  </Accordion>
</AccordionGroup>
