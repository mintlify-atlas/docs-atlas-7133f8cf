---
title: Selection sort
description: Simple algorithm that repeatedly selects the minimum element
---

Selection sort divides the array into a sorted and unsorted portion. It repeatedly finds the minimum element from the unsorted portion and places it at the end of the sorted portion.

## How it works

1. **Find minimum** - Scan the unsorted portion to find the minimum element
2. **Swap** - Swap the minimum with the first unsorted element
3. **Expand sorted region** - The sorted portion grows by one element
4. **Repeat** - Continue until all elements are sorted

<Note>
  Selection sort is notable for making the minimum number of swaps - at most n-1 swaps for an array of length n. This makes it useful when write operations are expensive.
</Note>

## Complexity

<CardGroup cols={3}>
  <Card title="Best case" icon="gauge-high">
    **O(n²)**
    
    Always scans entire array
  </Card>
  
  <Card title="Worst case" icon="gauge-low">
    **O(n²)**
    
    Same as best case
  </Card>
  
  <Card title="Space" icon="memory">
    **O(1)**
    
    In-place sorting
  </Card>
</CardGroup>

**Stability:** No - distant equal elements may be swapped

## Implementation

```typescript
import type { SortStep } from '$types/algorithm';

export function* selectionSort(arr: Uint8Array): Generator<SortStep, void, undefined> {
  const n = arr.length;

  for (let i = 0; i < n - 1; i++) {
    let minIdx = i;

    for (let j = i + 1; j < n; j++) {
      yield { type: 'compare', indices: [minIdx, j] };
      if (arr[j]! < arr[minIdx]!) minIdx = j;
    }

    if (minIdx !== i) {
      [arr[i], arr[minIdx]] = [arr[minIdx]!, arr[i]!];
      yield { type: 'swap', indices: [i, minIdx] };
    }

    yield { type: 'sorted', indices: [i] };
  }

  yield { type: 'sorted', indices: [n - 1] };
}
```

## Visualization steps

Selection sort uses three types of visualization steps:

### Compare
```typescript
{ type: 'compare', indices: [minIdx, j] }
```
Compares the current minimum with another element in the unsorted portion.

### Swap
```typescript
{ type: 'swap', indices: [i, minIdx] }
```
Swaps the minimum element with the first unsorted element. Only performed if `minIdx !== i`.

### Sorted
```typescript
{ type: 'sorted', indices: [i] }
```
Marks an element as placed in its final sorted position.

## Key features

<AccordionGroup>
  <Accordion title="Minimum swaps">
    Selection sort performs at most n-1 swaps, the minimum possible for a comparison-based sorting algorithm. This makes it efficient when write operations are much more expensive than reads (e.g., writing to flash memory or EEPROM).
  </Accordion>
  
  <Accordion title="Not adaptive">
    Unlike insertion sort or bubble sort, selection sort always performs O(n²) comparisons, even on already-sorted data. It gains no advantage from pre-existing order.
  </Accordion>
  
  <Accordion title="Not stable">
    Selection sort is not stable. For example, in `[5a, 3, 5b, 1]`, the first 5 will be swapped with 1, moving it past the second 5, changing their relative order.
  </Accordion>
  
  <Accordion title="Simple and predictable">
    The algorithm is extremely simple to implement and understand. It also has predictable performance - always O(n²) comparisons and at most O(n) swaps.
  </Accordion>
</AccordionGroup>

## Algorithm walkthrough

Here's how selection sort processes an example array `[3, 1, 4, 1, 5]`:

<Steps>
  <Step title="First pass">
    Find minimum in `[3, 1, 4, 1, 5]` → `1` at index 1
    
    Swap with position 0: `[1, 3, 4, 1, 5]`
  </Step>
  
  <Step title="Second pass">
    Find minimum in `[3, 4, 1, 5]` → `1` at index 3
    
    Swap with position 1: `[1, 1, 4, 3, 5]`
  </Step>
  
  <Step title="Third pass">
    Find minimum in `[4, 3, 5]` → `3` at index 3
    
    Swap with position 2: `[1, 1, 3, 4, 5]`
  </Step>
  
  <Step title="Fourth pass">
    Find minimum in `[4, 5]` → `4` at index 3
    
    Already in position, no swap: `[1, 1, 3, 4, 5]`
  </Step>
  
  <Step title="Done">
    Last element is automatically in position: `[1, 1, 3, 4, 5]`
  </Step>
</Steps>

## Performance characteristics

### Number of operations

| Operation | Count | Formula |
|-----------|-------|--------|
| Comparisons | Always (n² - n) / 2 | Θ(n²) |
| Swaps | At most n - 1 | O(n) |
| Array accesses | Θ(n²) | For comparisons |

### Comparison with other O(n²) algorithms

| Feature | Selection sort | Insertion sort | Bubble sort |
|---------|---------------|---------------|-------------|
| Comparisons | Always O(n²) | O(n) to O(n²) | O(n) to O(n²) |
| Swaps | At most n-1 | O(n²) | O(n²) |
| Adaptive | No | Yes | Yes |
| Stable | No | Yes | Yes |
| Online | No | Yes | No |

## When to use

<Check>
  Selection sort is useful when:
  - Write operations are expensive (flash memory, EEPROM)
  - Minimizing swaps is critical
  - Array size is very small (< 10 elements)
  - Simplicity is more important than performance
  - You need predictable performance regardless of input
</Check>

<Warning>
  Avoid selection sort when:
  - Working with large datasets
  - The array might be partially sorted (use insertion sort)
  - Stability is required
  - You need better than O(n²) performance
</Warning>

## Variations

<AccordionGroup>
  <Accordion title="Bidirectional selection sort">
    Also known as cocktail selection sort, this variation finds both the minimum and maximum in each pass, placing them at both ends of the array. This reduces the number of passes by half but doesn't improve the overall time complexity.
  </Accordion>
  
  <Accordion title="Heapsort">
    Heap sort can be viewed as an improved selection sort. Instead of linear search for the minimum, it uses a heap data structure to find the minimum in O(log n) time, achieving O(n log n) overall complexity.
  </Accordion>
</AccordionGroup>

## Why it's rarely used

Despite its simplicity, selection sort is rarely used in practice:

1. **Poor time complexity:** O(n²) even in the best case
2. **Not adaptive:** Doesn't take advantage of existing order
3. **Not stable:** Changes relative order of equal elements
4. **Better alternatives exist:** Insertion sort is almost always better for small arrays
